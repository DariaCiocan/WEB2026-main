📐 Diagrame UML — Todo List App

Arhitectura de ansamblu a aplicatiei web (React + Node.js/Express + JSON)

1. Diagrama Use Case

Use Case

Actorii sistemului si cazurile de utilizare disponibile. Utilizatorul interactioneaza cu aplicatia prin browser; serverul gestioneaza persistenta datelor.

2. Diagrama de Activitate — Adaugare Sarcina

Activity

Fluxul complet de la completarea formularului de catre utilizator pana la persistenta pe server si notificarea toast in UI.

3. Diagrama de Interactiune (Sequence)

Interaction

Schimbul de mesaje intre componentele sistemului la operatia de creare a unui task nou (scenariul de succes).

4. Diagrama de Componente

Component

Organizarea modulara a codului: componentele React, stratul API, serverul Express si persistenta JSON.

5. Diagrama de Deployment

Deployment

Distributia fizica a aplicatiei in productie: frontend pe Vercel, backend pe Render, comunicare prin HTTPS/REST.

Flux deployment: Push pe GitHub → Vercel rebuild automat frontend (< 1 min) → Render redeploy automat backend (~2 min). Frontend apeleaza backend-ul via variabila de mediu VITE_API_BASE_URL.
